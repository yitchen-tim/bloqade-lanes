Expand description

Architecture specification for a quantum device.

Describes the full hardware topology: geometry (words, sites, grids), bus connectivity, zones, and operational constraints. Can be loaded from JSON via from_json / from_json_validated, or constructed programmatically.

Fields§

§version: Version

Spec format version.

§geometry: Geometry

Device geometry (words and their site layouts).

§buses: Buses

Transport bus definitions (site buses and word buses).

§words_with_site_buses: Vec<u32>

Word IDs that have site-bus transport capability.

§sites_with_word_buses: Vec<u32>

Site indices that participate in word-bus transport.

§zones: Vec<Zone>

Logical zones grouping words for execution phases.

§entangling_zones: Vec<u32>

Zone IDs where entangling (CZ) gates can be performed.

§measurement_mode_zones: Vec<u32>

Zone IDs that support measurement mode. Must not be empty; the first entry must be zone 0.

§paths: Option<Vec<TransportPath>>

Optional AOD transport paths.

pub fn check_lane(&self, addr: &LaneAddr) -> Vec<String>

Check whether a lane address is valid.

Validates that the bus exists, word/site are in range, and the site/word is a valid forward source for the bus. In the lane address convention, site_id and word_id always encode the forward-direction source — the direction field only controls which endpoint is src vs dst, not which site/word is encoded.
